1. Introduction
This project develops a fine-grained beverage can classification system integrated into a ROS2 robot
perception pipeline. The main goal was to classify the exact beverage category during live robot oper-
ation instead of only detecting whether a can exists in the scene. Transfer learning was applied using
MobileNetV3-Large [1], implemented in PyTorch [2], together with OpenCV-based preprocessing utilities
[3].
Fine-grained classification is more difficult than standard object recognition because all beverage cans
share very similar cylindrical shapes. Most differences come from logo layout, colour combinations, and
text patterns [4]. The dataset covered 20 beverage-can categories including Coca-Cola Classic, Coca-Cola
Zero, Pepsi, Sprite, Fanta Orange, Monster, Red Bull, and others. The ImageNet-pretrained pipeline
[5] used 224×224 image resolution with Adam optimisation [6], random crops, flipping, rotation, and
colour jittering.
The ROS2 deployment pipeline subscribed to the RGB camera stream, published prediction outputs, and
controlled robot movement [7]. Three operating behaviours were implemented: fixed-position classifica-
tion, scan-based detection, and target-search behaviour (find, align, approach, stop). The live inference
pipeline used OpenCV contour-based region extraction together with MobileNetV3 classification. Main
deployment challenges included visual similarity between cans, robot-camera reflections, motion blur,
and unstable viewpoints.
2. Dataset Collection and Preparation
A custom dataset of 20 beverage-can categories was collected using smartphones and cameras across
different viewpoints, distances, and lighting conditions. Additional robot-camera images were later
collected directly from the live RGB stream during deployment testing. These additional images helped
expose the model to reflections, blur, and viewpoints not fully represented in the original dataset.

3.2 Region Detection and Cropping
OpenCV contour detection was used to identify candidate beverage-can regions before classification.
Passing cropped regions instead of the full RGB frame improved prediction stability by reducing unnec-
essary background information.
The contour-based preprocessing pipeline used grayscale conversion, Gaussian blur, Canny edge detection
[8], dilation, and aspect-ratio filtering to locate likely can regions. Additional padding was added around
cropped regions to preserve logo edges and improve classification quality under different viewing angles.

3.5 ROS2 Integration
The ROS2 deployment pipeline subscribed to the RGB camera stream, published prediction messages,
and controlled robot movement [7]. In scan mode, the robot rotated continuously while searching for
target cans. In target-search mode, the robot aligned detected cans to the centre of the frame, moved
forward, and stopped when the detected object occupied sufficient frame area or when a timeout occurred.

5. Robotic Deployment
5.1 Pipeline
The deployment stage connected the trained classifier directly to the physical robot. Live RGB frames
were converted using cv bridge [3], contour-based region extraction identified likely can regions, cropped
images were resized to 224×224, and MobileNetV3-Large predicted the final category label.

5.2 Deployment Modes
Three operating modes were implemented:
1. Fixed-position classification — the robot remained stationary while the camera stream and classifier
were tested separately.
2. Navigation-based scan mode — the robot rotated continuously while classifying beverage cans from
the live camera stream.
3. Target-search and stop — the robot cycled through SEARCH, ALIGN, APPROACH, and ARRIVED
states until the target object occupied sufficient frame area.
Table 10: Target-search state machine
State Behaviour
SEARCH Rotate while scanning for target class
ALIGN Centre detected target in frame
APPROACH Move forward while steering toward target
ARRIVED Stop robot movement
5.3 Initial Problems and Resolution
Early deployment produced instability because of camera-angle mismatch, background distraction, reflec-
tions, and motion blur. Additional robot-camera images were added into the dataset, crop preparation
was improved, and fine-tuning was repeated using deployment-style images.

6. Discussion
What worked well:MobileNetV3-Large with ImageNet transfer learning learned beverage-can logo
and colour patterns effectively [1, 5]. The crop-based classification pipeline and robot-camera fine-tuning
contributed strongly to deployment stability.
What was difficult:Visually similar class pairs such as Coca-Cola Zero versus Coca-Cola Classic
remained difficult. Robot deployment also introduced motion blur, reflections, cluttered backgrounds,
and unstable viewpoints. Contour detection occasionally struggled under difficult lighting and reflective
surfaces.
Limitations:The custom dataset could not fully represent every deployment environment. Augmenta-
tion could not completely replicate real robot-camera blur and reflections. Pipeline stability depended
on both contour-based region extraction and CNN classification performing reliably together.
7. Conclusion
This project successfully developed a fine-grained beverage-can classifier integrated into a ROS2 robot
deployment pipeline. Starting from a custom 20-class dataset, the system progressed through preprocess-
ing, contour-based crop preparation, transfer learning, robot-camera fine-tuning, and live deployment.
The final system achieved 97.20% accuracy on 1,927 test images and 100% accuracy on the 3-class target-
search subset. The deployment demonstrated fixed-position classification, rotating scan behaviour, and
target-search navigation under real operating conditions.
The project showed that strong offline accuracy alone is insufficient for robot perception. Crop-based
preprocessing, staged fine-tuning, and ROS2 integration were necessary for practical deployment stability.
Future improvements may include larger robot-camera datasets, stronger contour filtering, depth-based
navigation, and cross-frame tracking methods such as SORT [9].
